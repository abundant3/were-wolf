# 狼人杀行动流程

## 白天

1. **白昼降临 (day-announce)**
   - 系统公布昨夜死亡信息（或平安夜）
   - 检查胜负条件，若有人获胜则进入 game-over
   - 处理死亡触发（猎人/狼王开枪、恋人殉情），每次触发后重新检查胜负
   - 首夜死亡的角色可以发表遗言（用户有输入框，AI自动生成）
   - 遗言完毕后显示"轮流发言"按钮，用户点击进入发言

2. **固定发言 (day-speech)**
   - 所有存活玩家按座位顺序依次发言
   - 用户发言：无时间限制，有聊天输入框，点击"发言完毕"结束
   - AI发言：30秒超时，超时显示"超时无响应"；发言时显示"xxx发言中..."指示器，底部按钮呼吸灯状态
   - 发言完毕后直接进入投票阶段

3. **投票环节 (day-vote)**
   - 弹窗显示所有存活玩家按钮，用户点击投票（可弃权）
   - 投票后弹窗显示"你投了 XXX，等待其他玩家投票..."
   - AI玩家依次投票（间隔1秒避免API并发冲突）
   - 投票结果只显示每人被投多少票，不显示谁投了谁
   - 票数最高者出局（平票则无人出局）
   - 出局者触发遗言和死亡技能（白痴翻牌、猎人/狼王开枪）
   - 投票完毕后自动进入下一夜

## 夜晚

行动顺序（NIGHT_ORDER）：丘比特(仅首夜) → 守卫 → 狼人 → 女巫 → 预言家 → 猎人 → 白痴(仅首夜)

- 每个阶段自动跳过已死亡或不在场的角色
- 用户是当前角色时直接弹出操作面板，无需额外点击
- AI角色自动行动，用户看到"等待XX行动..."
- 所有夜间行动完成后进入 resolveNight 结算

## 游戏流程总览

```
idle → night(start) → night[各角色依次行动] → night(done) → resolveNight → day-announce → day-speech → day-vote → night(下一天) → ...
```

# 游戏机制

## 身份阵营

| 阵营 | 身份 | 说明 |
|------|------|------|
| 狼人 | 狼人 | 每晚与队友共同刀一人 |
| 狼人 | 狼王 | 参与刀人；被动致死（非毒杀）时可带走一人 |
| 狼人 | 白狼王 | 参与刀人；白天可主动自爆带走一人 |
| 好人(平民) | 村民 | 无特殊能力 |
| 好人(神职) | 预言家 | 每晚查验一人阵营 |
| 好人(神职) | 女巫 | 解药救人/毒药杀人，各限一次，不可同夜使用 |
| 好人(神职) | 守卫 | 每晚守护一人免受狼刀，不可连续守同一人 |
| 好人(神职) | 猎人 | 死亡时（非毒杀）可开枪带走一人 |
| 好人(神职) | 白痴 | 被投出时可翻牌免死，之后失去投票权 |
| 好人(神职) | 丘比特 | 首夜指定两人为恋人，一人死亡则另一人殉情 |

## 默认房间配置

12人局：4狼(狼人) + 4神(预言家/女巫/守卫/猎人各1) + 4村民

## 胜负判定

- **好人胜**：所有狼人出局
- **狼人胜**：所有好人出局（屠城规则）

## 夜间结算规则 (resolveNight)

1. 守卫守护的目标（铜水）不受狼刀伤害
2. 女巫解药可以救被狼刀的人（银水），但第0夜不能自救
3. **三重碰撞**：同一人同时被守卫、被狼刀、被女巫解药 → 该玩家死亡
4. 女巫毒药无视守卫，毒杀必定死亡
5. 守卫不能连续两晚守同一人

## 狼人密议机制

- 夜间狼人行动时，AI狼队友先依次发表战术讨论（30字以内）
- 用户是狼人时：显示讨论面板 + 聊天输入框，@提及列表只显示狼队友
- 讨论完毕后进入目标选择，AI狼根据讨论内容投票
- 投票统计：票数最高者被刀，平票则空刀

## 私密记忆系统

- 每个角色每次行动后会生成私密想法（temperature 1.0）
- 内容包括真实意图、是否打算欺骗、策略考虑
- 仅自己可见，后续行动时注入AI请求上下文
- 狼人知道自己队友身份，预言家知道自己查验记录

## AI上下文注入 (queryAIWithContext)

每次AI请求依次注入：
1. 角色专属系统提示词
2. 房间信息（人数、身份配置）
3. 私密记忆（历史想法）
4. 近期事件（最近10条）
5. 公开聊天记录
6. 当前行动的用户提示词

# 各个身份的系统提示词

## 狼人 (werewolf) — runNightWerewolf

狼人、狼王、白狼王共享同一套夜间行动流程（密议+投票），均参与刀人环节。

### 密议阶段
**System:** `你是狼人，与{队友名}是队友。现在是夜间狼人密议环节，你们要讨论今晚的目标。{角色性格}`
**User:** `第{N}天夜晚。可刀目标：{列表}。{之前的讨论}请简短发表你的看法，30字以内。只回复讨论内容。`

### 投票阶段
**System:** `你是狼人，与{队友名}是队友。你们每晚选择一名非狼玩家杀害，也可以选择空刀。{角色性格}`
**User:** `第{N}天夜晚。可刀目标：{列表}。狼人密议记录：{讨论内容}请回复目标名字或"空刀"。`

## 狼王 (wolfking) — 夜间同狼人，死亡触发开枪

夜间行动与普通狼人完全相同。区别在于被动致死（非毒杀）时可带走一人，复用 runShootAbility。

## 白狼王 (whitewolfking) — 夜间同狼人，主动自爆

夜间行动与普通狼人完全相同。区别在于白天可随时主动自爆带走一人（被动死亡不可发动）。

## 村民 (villager) — 无夜间行动

无特殊能力，不参与夜间行动。白天发言和投票时使用通用提示词。

## 预言家 (seer) — runNightSeer

**System:** `你是预言家。每晚可以查验一名玩家的阵营（好人或狼人）。{角色性格}`
**User:** `第{N}天夜晚。存活可查玩家：{列表}。{之前查验记录}。请回复你要查验的玩家名字。`

查验结果存入 `gs.players[id].seerChecks`，格式 `{ targetId, result: 'good'|'wolf' }`。

## 女巫 (witch) — runNightWitch

**System:** `你是女巫。你有一瓶解药（可以救被狼杀的人）和一瓶毒药（可以毒杀一人），各限用一次。不能在同一晚同时使用两瓶药。{角色性格}`
**User:** `第{N}天夜晚。{今晚被杀者信息}{解药状态}{毒药状态}{不能自救提示}存活其他玩家：{列表}。请决定：回复"救{名字}"使用解药，"毒{名字}"使用毒药，或"跳过"不使用。`

限制：第0夜不能自救；解药和毒药已用完后不再提示对应选项。

## 守卫 (guard) — runNightGuard

**System:** `你是守卫。你的任务是每晚守护一名玩家使其免受狼人袭击。{角色性格}`
**User:** `当前是第{N}天夜晚。存活玩家：{列表}。{上次守护信息}。请选择你要守护的玩家名字，只回复名字。`

限制：不可连续两晚守同一人。上次守护信息会提示"你上一晚守护了{name}"。

## 猎人 (hunter) — runNightHunter（占位）+ runShootAbility（死亡触发）

夜间无实际行动（占位函数）。核心技能在死亡时触发：

**System:** `你是{name}（猎人），你死了。你可以开枪带走一名玩家，被带走者无遗言。{角色性格}`
**User:** `可射击目标：{列表}。回复目标名字或"不开枪"。`

触发条件：被投出、被枪杀（非女巫毒杀）。

## 白痴 (idiot) — runNightIdiot（占位）+ 翻牌免死

夜间无实际行动（占位函数，仅首夜）。核心技能在投票出局时触发：

- 用户：弹窗选择"翻牌免死（之后无法投票）"或"接受出局"
- AI：自动选择翻牌

翻牌后标记 `isIdiotRevealed = true`，失去投票权（`canVote = false`）但可继续发言。

## 丘比特 (cupid) — runNightCupid（仅首夜）

**System:** `你是丘比特。你在游戏开始时选择两名玩家成为恋人。恋人一方死亡时另一方也会殉情。{角色性格}`
**User:** `请选择两名玩家作为恋人，只回复两个名字用逗号分隔。存活玩家：{列表}`

恋人存入 `gs.lovers` 数组。死亡时触发殉情（`killPlayer` 中级联检查）。

# 通用提示词（所有身份共用）

## 白天发言 (generateAISpeech)

**System:** `你是{name}，身份是{身份}。当前是第{N}天的白天讨论环节。存活玩家：{列表}。{额外信息}{角色性格}请发表你的观点和推理，用中文发言，语气符合角色性格，50-100字。只回复发言内容，不要加引号或前缀。`

额外信息按身份注入：
- 狼人阵营：`你是狼人阵营。你的狼队友：{队友名}`
- 预言家：`你是预言家。查验记录：{记录}`

## 投票 (runDayVote)

**System:** `你是{name}。当前投票环节，请投出你认为的狼人。{角色性格}`
**User:** `可投票目标：{列表}。回复玩家名字或"弃权"。`

## @提及回应 (triggerMentionResponses)

**System:** `你是{name}，正在参与狼人杀的自由讨论。{角色性格}你的身份是{身份}。`
**User:** `有人对大家说："{消息}"，其中@了你。请简短回应，50字以内，符合你的性格。`

## 私密想法 (generateAndStoreThought)

**System:** `你是{name}，在狼人杀游戏中身份是{身份}。{角色性格}请简短写下你此刻的真实内心想法（1-2句话），包括你的真实意图、是否打算欺骗他人、策略考虑。这段内心独白只有你能看到。`
**User:** `当前是第{N}天的{夜晚/白天}。你刚完成了"{行动}"行动。{行动结果}请用中文写下你此刻的真实内心想法。`

每个角色每次行动后都会调用，temperature 1.0 鼓励创造性思维。

# 技术架构

- 单页应用，hash路由（`#/meeting`、`#/creating`等）
- 视图生命周期：`init()` / `destroy()` 模式
- IndexedDB持久化存储（`werewolf`数据库，version 2）
- `advancePhase` 状态机调度器，按阶段拆分为独立 handler 函数
- `waitForHumanAction()` Promise 模式等待用户操作
- `withTimeout()` 30秒超时（基于 `Promise.race`）
- API请求间隔1秒，避免并发冲突
